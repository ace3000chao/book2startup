## 📖 什么是《易经》Skills？

《易经》成书于约公元前11世纪，由伏羲创八卦、文王写卦辞、周公作爻辞、孔子撰十翼，是中华儒家首经。

传统对《易经》的解读多停留在"算命"层面，但这本书**本质上是一套治国理政的方法论**——文王用它规划灭商大计，周公用它制定礼乐制度，孔子用它教导帝王。

本仓库将《易经》中的**可执行决策框架**提炼成13个 AI Agent Skills，让现代人可以直接调用这些千年智慧来解决实际问题。

## 🧩 效果示例

### 示例1：该不该等？

**用户问**：有个创业机会，条件看起来不错，但我总觉得还没准备好，要不要现在就开始？

**Skill 激活**：`shi-ji-deng-dai` 时机等待法

**执行结果**：
```
判断框架：需卦五层次
├── 郊（安全区）→ 耐心蛰伏
├── 沙（开始有杂音）→ 坚守
├── 泥（危险临近）→ 谨慎应对
├── 血（陷入险境）→ 冷静脱身
└── 饮食（安顿下来）→ 可以行动

结论：你现在处于"泥"的阶段——危险临近但还有时间。
建议：
1. 不要全面投入，但也不要完全放弃
2. 列出"时机成熟的标志"，达到60%就局部尝试
3. 设置止损线（如果X条件出现就撤）
```

### 示例2：成功后怎么守？

**用户问**：公司去年业绩很好，今年要怎么守住成果？

**Skill 激活**：`shou-cheng-jie-tan` 守成戒贪法

**执行结果**：
```
核心警示：亢龙有悔 — 成功后盲目扩张必有灾祸
节卦原则：节以制度，不伤财不害民

守成三原则：
1. 设定边界（当位以节）— 今年哪些事绝对不做？
2. 控制节奏（不伤财不害民）— 扩张前先问：会不会透支团队？
3. 警惕"苦节" — 过度节约同样是病，守成不是抠门

反例警示：商纣王—盛世时沉湎于酒，最终亡国
```

### 示例3：团队决策总是一个人说了算

**Skill 激活**：`yong-jiu-qun-long` 用九·见群龙无首

**执行结果**：
```
易经原文："见群龙无首，吉" — 群龙无首才是吉祥的

用九原则：
• 不做绝对权威，让各方力量自然发挥作用
• 对集体计划保持"观察、理解"，不强行主导
• 不出头告发（无首）= 不独断专行

执行步骤：
1. 重大决策前，先让每个人书面提交意见
2. 你最后发言，但只补充、不覆盖
3. 决策文件上，不要写"批准人"，写"集体决策"
```

##  📚 应用案例

问：很多创业者觉得“AI很强但是我就是用不上”，请问怎么办？

答：（我的openclaw创业导师Agent）

### 一、问题本质诊断

#### 核心判断

创业者的困境本质是：**"AI很强"（看到了山天大畜的"辉光"），但"我的积累不够"（还没准备好发射）**。

这不是"AI没用"的认知问题，而是**积累路径不清**的执行问题——

| 表现 | 真正原因 |
|------|----------|
| "AI很厉害，但不知道用在哪" | 认知积累有，场景积累无 |
| "学了很多AI工具，但还是用不上" | 工具积累有，流程积累无 |
| "AI能帮我吗？真的可以吗？" | 信息积累有，验证积累无 |

**结论**：他们在"大畜卦"的积累期，还没有到"发射"的时机。不是AI没有价值，是他们的应用积累还没到位。

### 二、厚积薄发法 · 五维积累自测表

#### 使用方法

让创业者对照以下五个维度自评（0-10分）：

| 维度 | 核心问题 | 自评（0-10） | 目标分 |
|------|----------|-------------|--------|
| **认知** | 我知道AI能做什么、不能做什么？ | ? | ≥6 |
| **工具** | 我有可用的AI工具账号、熟练使用至少1个？ | ? | ≥5 |
| **场景** | 我找到至少1个可以用AI提效的业务场景？ | ? | ≥5 |
| **流程** | 我的工作流程是否相对稳定、可复制？ | ? | ≥5 |
| **团队** | 有人可以一起研究、试错、互相提醒？ | ? | ≥4 |

#### 判停条件

- **五维全部 < 5**：还在积累早期，不要急着找"AI变现捷径"，先把五个维度逐个建立
- **3个以上维度 ≥ 5**：可以局部尝试，用小场景验证效果
- **五维全部 ≥ 6**：进入发射期，考虑扩大应用范围

### 三、障碍根源识别

积累不足有两类根本原因，用这个问题快速判断：

> **"你现在最想用AI解决的一个具体问题是什么？"**

| 回答 | 障碍类型 | 应对策略 |
|------|----------|----------|
| 答不上来/很模糊 | **方向型不足** | 先定方向，再谈积累（旅卦警示：方向不正有眚） |
| 能说出1-2个具体场景 | **时间型不足** | 正常积累期，耐心等待、厚积薄发 |

#### 方向型不足的警示

旅卦曰："其匪正有眚，不利有攸往"——方向不对，贸然行动必有灾祸。

如果创业者一直在"学AI"但没有明确的应用方向，要先停下来明确方向，否则积累越多越迷失。

#### 时间型不足的应对

大畜卦曰："刚健笃实，辉光日新"——积累不是停滞，而是在扎实基础上每日精进。

方向明确但觉得"还没准备好"是正常的，这是时间型积累期，需要耐心。

### 四、聚焦最小可用场景

厚积薄发**不是**"什么都准备好再动"，而是**找到一个最小的、可以验证价值的场景开始行动**。

#### 操作步骤

1. **选一个场景**：从日常工作中找最重复、最耗时间的一个环节
   - 示例：营销文案撰写 / 客服回复 / 数据整理 / 竞品分析 / 朋友圈发布

2. **只做一个**：先跑通一个场景，不要一开始就想"AI改造全公司"

3. **验证价值**：用"AI介入后，节省了多少时间？效果有没有打折扣？"来判断

#### 场景选择参考

| 场景 | 适合人群 | 验证周期 |
|------|----------|----------|
| 客服话术生成 | 有客服团队 | 1-2周 |
| 朋友圈/小红书文案 | 做私域运营 | 1周 |
| 竞品分析报告 | 做市场调研 | 2-3周 |
| 合同/协议初稿 | 商务谈判 | 2-4周 |
| 数据整理汇总 | 运营/财务 | 1周 |

### 五、90天积累里程碑

| 时间节点 | 里程碑目标 | 验证方式 |
|----------|-----------|----------|
| **第30天** | 熟练使用1-2个AI工具，形成自己的"主战工具" | 能独立完成1个场景的AI辅助 |
| **第60天** | 在1个业务场景中验证AI提效，形成可复制的流程 | 该场景节省时间 ≥ 30% |
| **第90天** | 总结该场景的AI使用SOP，考虑扩展到其他场景 | SOP文档化、可传授给他人 |

#### 复盘节点

- **第30天**：团队复盘——工具是否选对？场景是否合适？
- **第60天**：成果复盘——AI提效是否真实？流程是否可复制？
- **第90天**：总结复盘——形成什么方法论？下一步怎么扩？

### 六、指导者操作手册

#### 场景启动（30分钟）

1. **五维自测**（5分钟）：每人填写五维积累表
2. **场景聚焦**（10分钟）：讨论选定一个最小场景
3. **90天承诺**（15分钟）：书面写下第30/60/90天的里程碑

#### 复盘节奏

| 复盘节点 | 时长 | 内容 |
|----------|------|------|
| 第30天 | 15分钟 | 工具熟练度检查、场景是否需要调整 |
| 第60天 | 30分钟 | 提效数据验证、SOP成形 |
| 第90天 | 60分钟 | 全面复盘、方法论沉淀、下一步规划 |

### 七、易经原文摘要（供指导者参考）

#### 大畜卦（山天大畜）

> **"刚健笃实，辉光日新"**

- 刚健笃实：根基要刚强扎实，不能急躁冒进
- 辉光日新：积累不是停滞，而是持续迭代、每日精进

#### 旅卦（火山旅）

> **"其匪正有眚，不利有攸往"**

- 方向不正会招来灾祸
- 时机未到、准备不足时贸然行动，必然失利

#### 两卦合观

**积是基础，发是结果**。没有足够的积累，发出去的力量就是虚的；但积累不是为了永远不动，而是在积累中寻找合适的发射窗口。

核心是区分**"真正准备好"**和**"只是急切想动"**。

## 🔧 这些 Skills 是怎么生成的？

本仓库使用 [book2skill](https://github.com/kangarooking/cangjie-skill) 流水线从《易经》原著蒸馏而成。

### 蒸馏流程（6阶段）

```
阶段0: 整书理解 → 读完全书，提炼主旨、骨架、关键术语
   ↓
阶段1: 五路并行提取 → 5个Agent同时提取：
   • 框架提取器（思维模型）
   • 原则提取器（行为准则）
   • 案例提取器（历史案例）
   • 反例提取器（失败模式）
   • 术语提取器（核心概念）
   ↓
阶段1.5: 三重验证筛选
   • V1 跨域验证：在≥2个独立语境有佐证？
   • V2 预测力：能推导书中未明说的新问题？
   • V3 独特性：不是"任何聪明人都会说的常识"？
   ↓
阶段2: RIA++构造skill
   • R — 原文引用
   • I — 方法论骨架
   • A1 — 书中的应用案例
   • A2 — 触发场景（最关键）
   • E — 可执行步骤
   • B — 适用边界
   ↓
阶段3: Zettelkasten链接 → 识别skill间依赖/对比/组合关系
   ↓
阶段4: 压力测试 → 用测试prompt验证触发精准度
```

### 验证数据

- **候选总数**：65条
- **通过验证**：43条（66%通过率）
- **最终产出**：13个可执行 Skills
- **质量门**：全部通过 V1/V2/V3 三重验证

## 🚀 如何使用？

### 前置要求

- [OpenClaw](https://github.com/openclaw/openclaw) 已安装
- 或任何支持 Skill 格式的 AI Agent 框架

### 安装方式

#### 方式1：克隆整个仓库

```bash
git clone https://github.com/ace3000chao/book2startup.git
cd book2startup/易经skills/yijing-skills
```

将 `yijing-skills` 目录添加到你的 Skill 搜索路径。

#### 方式2：单独安装某个 Skill

只想要"时机等待法"？直接复制对应文件夹：
```bash
git clone https://github.com/ace3000chao/book2startup/易经skills.git
cp -r yijing-skills/shi-ji-deng-dai ~/your-skills/
```

### 在对话中使用

当你遇到符合描述的场景时，直接说出来，AI 会自动匹配适合的 Skill：

| 你可以说 | 自动激活 |
|----------|----------|
| "该不该等这个时机？" | `shi-ji-deng-dai` |
| "成功了要怎么守？" | `shou-cheng-jie-tan` |
| "陷入困境了该进还是退？" | `po-ju-jie-kun` |
| "我要变革但推不动" | `bian-ge-chuang-xin` |
| "团队总是一个人说了算" | `yong-jiu-qun-long` |

### 在代码中调用

```javascript
// 以 OpenClaw 为例
const skill = await agent.loadSkill('yijing-skills/shi-ji-deng-dai');
const result = await skill.execute({
  userQuestion: "有个创业机会，要不要现在就开始？"
});
```

### Skill 的 Trigger 信号

每个 Skill 都有自己的触发条件，详见各文件夹下的 `SKILL.md` 文件的 `A2 触发场景` 章节。

## ⚠️ 重要说明

1. **这不是占卜**：本仓库不涉及算命、风水等迷信内容，所有 Skill 都是可执行的决策框架

2. **原文理解**：本仓库对《易经》的解读基于[《易经并非天书》]的学术研究，与传统民间解读有本质区别
