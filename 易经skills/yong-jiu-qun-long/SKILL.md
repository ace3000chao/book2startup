# 用九，见群龙无首，吉

## R — 原文 (Reading)

> "用九，见群龙无首，吉。"
> "乾元用九，天下治也。"
> — 《易经》乾卦·用九

---

## I — 方法论骨架 (Interpretation)

"群龙无首"不是没有领袖，而是让多条龙各展所长，没有哪条龙处于绝对控制地位。这是最理想的组织状态——每个人都是龙，都有决策权，整体却和谐有序。

乾元用九，天下治：天的元气发挥到极致（用九），天下才能大治。应用到创业决策中：
- **警惕"首"的诱惑**：创业者天然倾向"我来决定"，但越重要的决策越需要集体智慧
- **群龙并起**：让核心团队成员各自表达见解，形成真正的讨论而非走过场
- **无首的机制**：建立让所有人都能发言、最终决策基于共识而非权威的决策流程
- **吉的来源**：这种集体决策之所以"吉"，是因为它调动了组织整体智慧，降低了个人盲点带来的风险

本质是用"机制"替代"意志"，让决策质量超越个人能力边界。

---

## A1 — 书中的应用 (Past Application)

### 案例1: 周公还政成王

- **问题**: 周武王去世后，年幼的成王继位，周公旦作为摄政王实际掌控朝政七年，权力达到顶峰
- **方法论的使用**: 周公没有选择"首"的路线，而是在成王成年后还政于王，让周朝回到群龙并立的格局，而非一人独大
- **结论**: 周公以"无首"为法，还政成王，建立起真正的集体传承机制
- **结果**: 周朝政治稳定，为八百年基业奠定制度基础；周公因此被尊为"元圣"

### 案例2: 华为轮值CEO制度

- **问题**: 任正非作为华为创始人，如何避免个人决策局限性影响企业长期发展
- **方法论的使用**: 华为建立轮值CEO制度，让不同领导者轮番主持决策，打破"一龙首"模式
- **结论**: 用机制替代个人权威，组织智慧超越个体
- **结果**: 华为在多次重大决策中避免了个人判断失误，成为全球科技领导者

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 创业者独自面对重大决策，有强烈的个人倾向，并准备直接拍板（"我想这么做"）
2. 团队讨论变成一言堂，其他成员附和创业者意见，缺乏真正的异议
3. 创业项目进入关键转折期（融资、战略调整、核心人员整合），决策风险极高

### 语言信号

- "这事我说了算"
- "按我说的办就行了"
- "你们有什么意见吗？"（其实内心已有定论）
- "我觉得这个方向是对的，不需要再讨论了"
- "我是创始人，失败了也是我负责"

### 与相邻 skill 的区分

- 与 ce01（亢龙有悔）的区别: ce01强调"过度亢进必有悔"，本skill强调"独断决策的系统性风险"；本skill更侧重决策机制层面
- 与 f19（盈亏警示法）的区别: f19是风险预警，本skill是决策方式；两者可配合使用

## E — 可执行步骤 (Execution)

1. **暂停独断，引入异见**
   - 完成标准: 在做出最终决定前，专门安排一次"反对意见征集"，明确询问"谁反对？为什么？"
   - 判停条件: 若团队≤3人，跳到步骤3

2. **建立集体决策机制**
   - 完成标准: 将原决策问题拆解为3个以上子问题，让不同成员分别负责研究和提出方案，最终合并
   - 判停条件: 若成员拒绝参与讨论，记录原因后跳到步骤4

3. **设置决策缓冲期**
   - 完成标准: 对重大决策设立"24-72小时冷静期"，期间不正式宣布结论，允许任何人提出新信息
   - 判停条件: 若情况紧急（≤24小时需决策），记录风险后记录放弃本skill的原因

4. **记录分歧，达成共识**
   - 完成标准: 最终决策形成文字，说明"一致同意"还是"多数同意但存在以下分歧"，分歧内容一并记录
   - 判停条件: 若最终仍是"创始人独断"，则在记录中标注"本决策未使用集体决策机制，风险自担"

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 紧急危机情境（公司面临生死存亡的即时威胁，≥24小时必须决断），此时需要快速决策而非民主讨论
- 团队规模极小（≤3人），缺乏集体决策的基础，另类讨论反而制造尴尬
- 所涉决策属于创始人个人专业领域且团队完全缺乏相关经验——此时创始人的判断力确实是团队最优资产

### 作者在书中警告的失败模式

- "用九"的反面是"只用一"，即完全依赖单一决策者；历史上无数王朝和企业的覆灭都源于此
- 群龙无首之所以吉，是因为它调动了所有人的智慧；假装民主实际独断，比直接独断更危险

### 作者的盲点 / 时代局限

- 封建宗法背景：乾卦的"群龙无首"预设了贵族共同体的存在，现代创业企业需要主动构建这种共同体
- 德治乌托邦假设：书中假设参与者都是"龙"（有德有才），现实中团队成员能力参差，需要领导者在使用本skill时承担更多引导责任

### 容易混淆的邻近方法论

- ce01（亢龙有悔）：更强调"过度"的危害，本skill强调"机制缺失"的危害；两者互补
- p06（或跃在渊）：强调个体蓄势待发，本skill强调集体决策；方向上一个是个体进击，一个是组织收敛
