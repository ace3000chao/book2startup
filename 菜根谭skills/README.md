## 这套skills能解决什么问题

在AI时代，我们擅长用「怎么做」来问问题，却渐渐忘了先问「应不应该做」「值不值得做」「要不要退这一步」。

《菜根谭》是我是洪应明写的格言集，涵盖**修身、处世、名利、生死、心性觉醒**五大主题。我是通过cangjie-skill蒸馏法，把这47条古典智慧，转化成了**AI时代可调用的47个Skills**。

## 效果示例

### 示例1：谈判僵局场景

**用户prompt：**
> 我在跟一个投资人谈判，条款已经僵持了两周，对方态度很强硬，再争下去就要撕破脸了，要不要让一步？

**Skill `f01` 激活后的输出：**
```
【退让一步决策框架】已激活

📊 当前局面诊断：谈判僵持（资源型+关系型混合）
代价评估：
  - 不让的代价：关系破裂 + 机会丧失 + 时间浪费
  - 让的代价：短期条款让步

💡 建议方案（局部退让，保留核心）：
  1. 在非核心条款上让步（如付款周期）
  2. 在核心条款上坚持（估值/股权比例）
  3. 用「下次优先权」换取对方的面子

⏱️ 时机判断：现在是退让的最佳时机（情绪尚未完全激化）
```

### 示例2：团队管理场景

**用户prompt：**
> 团队里有个人经常在背后说人坏话，其他成员都开始学他，氛围越来越差，怎么处理？

**Skill `p12` + `p08` 激活后的输出：**
```
【不恶小人待以礼原则 + 和气致祥原则】已激活

🔍 诊断：这不是道德问题，是「风气传导」问题

📌 核心洞察：
  「风能吹灭蜡烛，也能让火越烧越旺」
  风气坏了，管理者是第一个受害者，也是第一个责任人

💡 处理步骤：
  1. 不当面斥责（会让小人更隐蔽，团队更分裂）
  2. 私下单独沟通（给台阶，不给公开舞台）
  3. 在公开场合表扬「不背后说坏话」的行为（树正例）
  4. 建立「有话当面说」的团队规范

⚠️ 边界：不要让小人有「被欺负」的感觉，否则他会博同情
```

## 这些Skills是怎么生成出来的

### 方法论：RIA-TV++ 五阶段流水线

```
阶段0: 整书理解 → BOOK_OVERVIEW.md
阶段1: 5个并行extractor → 117条候选单元
阶段1.5: 三重验证筛选 → 47条通过，15条降级
阶段2: RIA++构造 → 47个SKILL.md
阶段3: Zettelkasten链接 → 关系网络 + INDEX.md
阶段4: 压力测试 → 47个test-prompts.json
```

**三重验证**是质量核心，确保每个skill：
- **V1跨域**：书中至少2个独立语境有佐证
- **V2预测力**：能推导出书里没明说的新问题答案
- **V3独特性**：不是「任何聪明人都会说的常识」

### 工具：cangjie-skill（book2skill元skill）

> 📖 [cangjie-skill源码](https://github.com/kangarooking/cangjie-skill)

cangjie-skill是我是kangarooKing开发的「把书蒸馏成skills」的元skill。它的核心思想是：

**「现有读书方法论都是为人类读者蒸馏，不是为AI执行者蒸馏。」**

因此，RIA-TV++的所有扩展（V1/V2/V3/E/B/test-prompts）都是为了解决「目标读者从人变成AI」带来的新问题。

这套《菜根谭》skills就是用cangjie-skill流水线生成的。

## 如何安装和使用

### 方式一：直接复制到你的Skills目录

```bash
# 1. 克隆仓库
git clone https://github.com/ace3000chao/book2startup.git
cd book2startup

# 2. 进入菜根谭skills目录
cd 菜根谭skills

# 3. 查看所有skills
ls skills/

# 4. 复制你需要的skill到你的AI工具skills目录
cp -r skills/f01 /你的skills目录/
```

### 方式二：使用book2skill流水线重新蒸馏

如果你想修改或扩展：

```bash
# 1. 安装cangjie-skill
git clone https://github.com/kangarooking/cangjie-skill.git

# 2. 准备书籍文本（PDF/TXT）
pdftotext 你的书.pdf 书本文本.txt

# 3. 运行book2skill流水线
python run_book2skill.py --book 书本文本.txt --book-title "书名" --author "作者"

# 4. 查看产出
ls books/书名/
```

### 在AI助手中调用

**方式1：在对话中直接引用**
```
请用【退让一步决策框架】分析这个谈判场景：
[粘贴你的场景描述]
```

**方式2：让AI自动选择**

在你使用的AI助手中配置skill目录，AI会在合适的时机自动选择对应的skill。
