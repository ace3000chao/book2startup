# 守正安分远祸原则

## R — 原文 (Reading)

> "趋炎附势之祸，甚惨亦甚速；栖恬守逸之味，最淡亦最长。"
>
> — 洪应明, 《菜根谭·守正安分 远祸之道》

---

## I — 方法论骨架 (Interpretation)

1. **趋炎附势的祸患特点：惨 + 快。** 依附权贵可能一时风光，但灾难来临时又快又惨。
2. **栖恬守逸的滋味特点：淡 + 长。** 安贫乐道、清静自守的滋味平淡，但持久。
3. **"栖恬守逸"不是躺平，是不攀附。** 保持自己的节奏和原则，不因为利益而依附权贵。
4. **核心逻辑：依附权贵是把命运交给别人，权势倒了，你也倒。** 守正安分是把命运放在自己手里。
5. **这是一个风险管理原则：不是看谁赚得多，是看谁活得久。**

---

## A1 — 书中的应用 (Past Application)

### 案例: 洪应明论"趋炎附势之祸甚惨亦甚速"

- **问题**: 依附权贵真的能获得好处吗？代价是什么？
- **方法论的使用**: 洪应明指出依附权势的祸患"甚惨亦甚速"——灾难来得又快又狠
- **结论**: 依附权贵是一笔高风险交易——短期可能获利，但一旦权势崩塌，代价是毁灭性的
- **结果**: 历史上依附权贵的人，很少有好下场

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要 this skill?

1. **面临"要不要攀附权贵"的抉择时** — "这个机会是依附一个权贵，我要不要？"、"我能通过这个关系获得更多资源"
2. **担心"出头椽子先烂"时** — "我最近太出头了，会不会遭殃？"、"我是不是该低调点？"
3. **看到别人靠关系获得好处，自己要不要跟随时** — "他这样攀关系，我不跟会不会落后？"、"靠关系真的好吗？"
4. **在"快速成功"和"安全长久"之间权衡时** — "我想快速成功，但这条路有没有风险？"

### 语言信号 (用户的话里出现这些就应激活)

- "要不要跟这个权贵？"
- "出头会不会遭殃？"
- "我该不该低调点？"
- "他靠关系上去的，我要不要学？"
- "快速成功和长久安全哪个更重要？"
- "我最近太高调了，会不会有问题？"

### 与相邻 skill 的区分

- 与 `p01栖守道德` 的区别: p01强调"身后之身"（身后名声），p28强调"远祸"（避开灾祸）；p01格局更高（名声），p28更实用（安全）
- 与 `p11退让一步加让三分` 的区别: p11是具体进退策略，p28是"不依附权贵"的原则；p11是战术，p28是战略

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **识别用户面临的是"权贵依附"的诱惑**
   - 完成标准: 用户明确这个机会涉及攀附权贵

2. **用"趋炎附势之祸甚惨亦甚速"测试：如果这个权贵倒了，你会怎样？**
   - 完成标准: 用户意识到依附关系的风险

3. **用"栖恬守逸之味最淡亦最长"测试：如果不依附，你能接受"淡"吗？**
   - 完成标准: 用户理解"平淡而长久"的价值

4. **权衡：如果依附，潜在收益X，潜在风险Y；如果不依附，潜在收益A，潜在风险B**
   - 完成标准: 用户做出知情的决定

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **需要建立正当关系网络的场景**：正常的合作关系、行业交流不是"趋炎附势"
- **战略进攻期需要借力时**：创业期可能需要借助资源，但不能迷失方向
- **对方是"贵人"而非"权贵"**：区分"帮助你的人"和"利用你的权贵"

### 作者在书中警告的失败模式

- 依阿权势之患（来自反例ce01）：开始为了利益接近权贵，为讨好权势而放弃原则，只看眼前利益不顾身后名节
- 机制：依附权贵是把自己的命运交给他人掌控，一旦权势崩塌，人格与名誉一同陪葬

### 作者的盲点 / 时代局限

- 现代社会的"关系网络"有时是必要的，不全是"趋炎附势"
- 完全不建立任何关系网络可能让人处于竞争劣势

### 容易混淆的邻近方法论

- **栖守道德(p01)**：p01是"坚守道德，宁可寂寞"，p28是"不依附权贵，避开灾祸"；p01格局更高（名声），p28更务实（安全）
- **退让一步(p11)**：p11是"进退有度"，p28是"不攀附"；p11是策略，p28是原则
