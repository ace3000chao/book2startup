# 苦中有乐得意生悲（福祸相依框架）

## R — 原文 (Reading)

> "苦心中常得悦心之趣，得意时便生失意之悲。"
>
> — 洪应明, 菜根谭·苦中有乐 得意生悲

---

## I — 方法论骨架 (Interpretation)

**顺境中藏悲的种子，逆境中藏乐的种子。**

这个框架提供了一种"预判事物演变"的推理能力：
1. **苦中有乐**：在逆境时，不过度悲观。要看到苦中有乐的可能——苦难往往是磨练，困境往往能激发出更强的生命力。
2. **得意生悲**：在顺境时，主动识别"乐极生悲"的风险点。成功越大，风险越大；巅峰之后往往是下坡。

**方法论本质：**
- 在任何状态下，都问自己一句："此刻的'极'是什么？它的反面什么时候会来？"
- 顺境时：主动识别"得意生悲"的风险点，提前防范，不要在成功时忘乎所以。
- 逆境时：主动寻找"苦中有乐"的种子，不轻易放弃，相信转化可能。

**一句话总结：** 福祸相依，得意时想失意，失意时想得意。

---

## A1 — 书中的应用 (Past Application)

### 案例 1: 塞翁失马焉知非福
- **问题**: 边塞老翁的马跑到了胡地——对普通人来说是"失"（倒霉）。
- **方法论的使用**: 老翁用"福祸相依"的框架看问题：失马可能是祸（丢了财产），但也可能带来更好的结果（马带回胡马，反而得福）。他不对任何单一事件做终局判断。
- **结论**: "苦中有乐"——失去一匹马这个"苦"中，可能藏着得马的"乐"。
- **结果**: 马带回胡马（因祸得福），但儿子骑胡马摔断腿（因福得祸）。老翁始终用"福祸相依"的框架看所有变化，不在得意时忘形，不在失意时绝望。

### 案例 2: 项羽的"得意生悲"
- **问题**: 项羽在巨鹿之战后达到军事巅峰，诸侯膝行而前，莫敢仰视——这是绝对的"得意"时刻。
- **方法论的使用**: 项羽没有识别"得意生悲"的风险点：他的成功让他更加刚愎自用，听不进范增的建议，不再审视自己的弱点。"得意"让他失去了危机意识。
- **结论**: "得意生悲"不是宿命，而是提醒：得意时如果忘形，风险就在积累。
- **结果**: 项羽在巅峰后不过四年，乌江自刎。他的失败不是突然的，而是从"得意时失去警觉"那一刻就开始了。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. **处于顺境时担心风险**：用户正处于事业高峰、关系顺利、赚钱容易的阶段，开始隐隐担心"这么好的状态能持续吗"。
2. **处于逆境时找不到希望**：用户正在经历失败、损失、低谷，觉得"一切都没有希望了"，需要找到"苦中有乐"的可能性。
3. **做重大决策前的风险评估**：用户要做一个重大决定（扩张、投入、结婚、离职），想评估这个决定可能带来的福祸转换。
4. **复盘自己的福祸轨迹**：用户在年度/季度复盘时，想看看自己过去一段时间是"苦中有乐"还是"得意生悲"，以便调整接下来的策略。
5. **看到他人成功/失败时想理解原因**：用户看到别人"一夜暴富"或"突然崩塌"，想理解福祸转换的逻辑。

### 语言信号 (用户的话里出现这些就应激活)

- "最近太顺了，感觉要出事"
- "我现在处于巅峰，但有点担心"
- "坏事发生了我觉得一切都完了"
- "我现在很苦，但不知道能不能熬过去"
- "为什么他这么成功却突然倒了"
- "为什么我这么倒霉，什么时候才能好起来"
- "我拥有的太多了，是不是该小心点"
- "我觉得现在的情况不会持久"

### 与相邻 skill 的区分

- 与 `f06（宠辱不惊情绪决策框架）` 的区别：f06 教人"得意不失态、失意不失志"的情绪管理；f10 教人"得意时看到风险、失意时看到机会"的预判能力。f06是情绪稳定，f10是风险预判。
- 与 `f07（见微知著守正待时）` 的区别：f07 讲"伏久者飞必高、开先者谢独早"的时机判断；f10 讲"苦中有乐、得意生悲"的福祸转化。f07是判断时机，f10是预判风险。
- 与 `f13（逆增上缘修行框架）` 的区别：f13 讲"逆境是修行的机会"（逆境中有成长）；f10 讲"逆境中可能有乐、顺境中可能有悲"（福祸相依）。f13是认知框架，f10是动态预判。

## E — 可执行步骤 (Execution)

当 skill 被激活后， agent应按以下步骤执行:

1. **确认用户当前的状态**
   - 完成标准: 明确说出"你现在处于苦（逆境）还是得意（顺境）"，并评估程度（1-10）。
   - 判停条件: 若用户处于极端状态（苦>8或得意>8），先让用户冷静，不进入下一步的深层分析。

2. **逆向推演福祸转换点**
   - 完成标准: 苦→问"这个苦中藏着什么乐的种子？"得意→问"这个得意中埋着什么悲的隐患？"
   - 判停条件: 若用户无法找到任何福祸转换的可能（如"这只有苦没有乐"），则引导其从"最小的事"开始寻找，如"这件事让你学到了什么"。

3. **制定"防悲/寻乐"的具体行动**
   - 完成标准: 得意时——列出"防止乐极生悲的三个具体措施"（如：不要在这个时候做重大决策、不要在这个时候加大投入、不要在这个时候得罪人）；苦时——列出"在苦中寻找乐的三个具体方向"（如：这件事教会了我什么、谁在这个过程中帮了我、这个经历以后能怎么用）。
   - 判停条件: 若用户处于极端困境（严重心理创伤），则简化步骤，聚焦于"先活过今天"，不宜做复杂的福祸推演。

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **用户正在经历急性危机**：如严重创伤、急性抑郁发作，此时"苦中有乐"可能被滥用为"你应该感谢苦难"的精神控制，对用户有害无益。
- **用户处于极端顺境并已出现躁狂症状**：此时"得意生悲"可能触发用户的防御性否认，让其无法接受任何风险提示。
- **用户正在为过去的事后悔**：f10是预判未来的工具，不是评估过去的工具。不应用来反复咀嚼"当初如果我没那么得意就好了"。

### 作者在书中警告的失败模式

- **快意不知回首（ce03）**：春风得意时不知见好就收，贪恋权位不肯急流勇退。表现为处于巅峰时愈发贪婪、不愿接受任何退让、享受众人吹捧不能自拔。这正是f10要预防的"得意生悲"的典型模式。
- **以短攻短以顽济顽（ce15）**：当众揭人短处、以愤怒厌恶对待别人的愚顽，恶性循环。人在"苦"中时更容易对他人产生怨恨，ce15提醒不要在逆境中用愤怒来处理问题。

### 容易混淆的邻近方法论

- **"福祸相依" vs "宿命论"**：福祸相依是"变化是必然的，你需要做好准备"；宿命论是"一切都是注定的，你做什么都没用"。f10是动态预判，不是静态宿命。
- **"苦中有乐" vs "美化苦难"**：苦中有乐是"苦难中有成长的可能"；美化苦难是"苦难是好事，你应该感谢"。前者是寻找可能性，后者是否认伤害。
