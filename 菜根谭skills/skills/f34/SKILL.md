# 事上敬谨 待下宽仁框架

## R — 原文 (Reading)

> "士君子处权门要路，操履要严明，心气要和易，毋少随而近腥膻之党，亦毋过激而犯蜂虿之毒。"
>
> — 洪应明, 事上敬谨 待下宽仁

---

## I — 方法论骨架 (Interpretation)

这是**管理者/当权者的处世四维框架**，核心是避免两个极端：

四个维度：

1. **操履严明**：原则清晰，规矩分明，不因人而异。对所有人都用同一个标准，不能对亲近的人放松，对不喜欢的人苛刻。

2. **心气和易**：态度温和，不盛气凌人。不因为自己有权力就高高在上，不对下属摆脸色。

3. **不近腥膻之党**：不与奸邪小人结盟。一旦与之为伍，就失去了道义的根基，终将被拖累。

4. **不激化与蜂虿之毒**：不主动激化与阴险小人的冲突。与恶人正面冲突往往两败俱伤，最好保持距离，不给他们伤害你的机会。

核心逻辑：管理者处于复杂的人际生态中，既要有清晰的原则（严明），又要有弹性的态度（和易）；既不能站错队（腥膻之党），也不能树敌过多（蜂虿之毒）。

---

## A1 — 书中的应用 (Past Application)

### 案例 1: 萧何月下追韩信
- **问题**: 韩信有大才但出身低微，萧何作为刘邦的重臣，如何在刘邦身边为韩信争取机会
- **方法论的使用**: 萧何对刘邦"操履严明"——始终以大局为重，不因私废公；对韩信等人才"心气和易"——愿意为他们争取机会
- **结论**: 好的管理者既能维护原则，又能包容人才
- **结果**: 韩信被拜为大将，助刘邦定鼎天下

### 案例 2: 项羽的失败——严而不和
- **问题**: 项羽本身有能力、有魅力，但团队最终离散
- **方法论的使用**: 项羽"操履严明"但"心气不和易"——待人刻薄（范增多次建议不被采纳），不能容忍不同意见
- **结论**: 只有严明没有和易，团队会失去人心
- **结果**: 韩信、陈平等人才纷纷离开，投奔刘邦，最终乌江自刎

### 案例 3: 刘邦的宽仁——和而不严的反面
- **问题**: 刘邦在某些时候过于宽容（对雍齿的背叛既往不咎）
- **方法论的使用**: 刘邦在"心气和易"上做得很好，但"操履严明"时有欠缺——有时候因私情而破坏了规矩
- **结论**: 只有和易没有严明，同样会出问题
- **结果**: 刘邦靠团队（张良/韩信/萧何）的集体智慧弥补了个人缺陷，但仍然出现了一些管理问题

### 案例 4: 曾国藩的平衡
- **问题**: 曾国藩处于晚清复杂的政治环境中，上面有朝廷，下面有湘军各级将领
- **方法论的使用**: 对朝廷"敬谨"——不显山露水，功高不震主；对下属"宽仁"——宽容有才者不好的一面，笼络人心
- **结论**: "敬谨"和"宽仁"在不同维度同时使用，而不是非此即彼
- **结果**: 功高震主却得以善终，是清代最成功的军事-政治领袖之一

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. **新晋管理者困惑**：刚当上领导，不知道如何拿捏对上的态度和对下属的态度
2. **团队管理困境**：对下属太严，团队氛围差、离职率高；对下属太松，下属不听话、工作没质量
3. **政治敏感性时刻**：面对复杂的办公室政治，不知道该站哪边、该保持什么距离
4. **对上关系处理**：不知道如何在领导面前既保持原则又不显得不服从

### 语言信号 (用户的话里出现这些就应激活)

- "我对下属太严厉了吗"
- "老板对我有看法我不知道怎么办"
- "我对下属已经很好了但他们还是不满意"
- "当了领导不知道怎么拿捏"
- "我该严还是松"
- "这个人是"小人"我不知道该怎么对待他"
- "我不喜欢这个人但他工作能力强，我不知道怎么和他相处"
- "上面有压力，下面有怨气，我不知道怎么处理"

### 与相邻 skill 的区分

- 与 `f04 中和处世决策框架` 的区别：f04是通用的性情判断（躁/刻/执），本框架是管理者专属的政治-人际平衡
- 与 `f33 不动声色应变框架` 的区别：f33是被动应对（被骗/被辱），本框架是主动构建管理者的人际策略

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **诊断当前失衡**
   - 完成标准: 用户对照四个维度自我评估：哪里强了（过于极端）？哪里弱了（过于忽视）？
     - 操履严明过强 → 对下属苛刻、规矩太多
     - 心气和易过强 → 对下属放纵、失去威信
     - 过于接近某一方 → 可能站错队
     - 过于激化冲突 → 树敌过多

2. **设定"动态平衡"基准**
   - 完成标准: 用户确定一个基准线（不是固定的比例，而是因时因事调整）：
     - 对上：敬谨但不谄媚
     - 对下：宽仁但有边界
     - 对"腥膻之党"：坚决不接近
     - 对"蜂虿之人"：保持距离但不主动激化

3. **执行具体场景测试**
   - 完成标准: 用户回忆最近一次管理困境，用"四维框架"复盘：我在哪一维出了问题？下次遇到类似情况，我应该在哪一维加强？

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 平级关系——本框架是专门针对上下级关系的管理框架
- 普通员工对领导——普通员工不需要"敬谨"（那是谄媚），也不应该对领导"宽仁"（那是错位）
- 把"宽仁"当成溺爱——宽仁不是纵容，是有底线的包容
- 把"严明"当成苛刻——严明是有标准，不是对人不留情面

### 作者在书中警告的失败模式

- ce05（富贵忌刻）："富贵家宜宽厚而反忌刻"——有权有势后反而刻薄待人，是最常见的领导者失败模式
- ce06（偏激为灾）："躁性者火炽，遇物则焚"——管理者过度情绪化是灾难
- ce14（器小禄薄）："鄙夫念头迫促"——管理者若心胸狭窄，只看眼前小利，必然失去人心

### 容易混淆的邻近方法论

- **f04（中和处世）**：两者都强调不走极端，但f04是通用的性情框架，本框架是管理者专属框架；f04适合所有性格问题，本框架只适合管理者
