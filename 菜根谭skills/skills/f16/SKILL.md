# 忘功念过 忘怨记恩框架

## R — 原文 (Reading)

> "我有功于人不可念，而过则不可不念；人有恩于我不可忘，而怨则不可不忘。"
>
> — 洪应明, 菜根谭·忘功念过 忘怨记恩

---

## I — 方法论骨架 (Interpretation)

**不对称记账——对自己的付出记少，对别人的恩惠记多。**

这个框架提供了一种人际恩怨的"记账原则"：
1. **我付出了，不要记**：你对别人好，不要念念不忘，否则这个"恩"会变成你期待回报的筹码。
2. **我有过失，必须记**：你伤害了别人，不要假装没发生，要记住并承担责任。
3. **别人对我有恩，记住**：别人帮助了你，要记住并感恩，不要把别人的好当成理所当然。
4. **别人对我有怨，忘掉**：别人伤害了你，不要记仇，让它过去，否则你的内心会被怨恨填满。

**核心逻辑：**
- 对自己：严格——功少记，过多记。
- 对别人：宽松——恩多记，怨少记。
- 这不是道德要求，而是"让自己内心更轻松"的策略——记太多怨恨会让你成为囚徒。

**一句话总结：** 记住该记住的，忘掉该忘掉的。

## A1 — 书中的应用 (Past Application)

### 案例 1: 诸葛亮的"忘功"
- **问题**: 诸葛亮一生为蜀汉鞠躬尽瘁，辅佐刘备、刘禅两代，功勋卓著。但他从未"念功"——他从未以"我对蜀汉有功"来要求特殊待遇或回报。
- **方法论的使用**: 诸葛亮在《出师表》中说的是"此臣所以报先帝而忠陛下之职分也"——他说的是"报恩"，不是"记功"。他把自己的一切付出定义为"应该做的"，不是"需要被记住的"。
- **结论**: "忘功"不是否定自己的付出，而是"不把付出变成期待回报的筹码"。
- **结果**: 诸葛亮死后，蜀汉百姓追思，千古流芳。"忘功"的人，反而被历史记住。

### 案例 2: 舜的"忘怨"
- **问题**: 舜的父亲和弟弟多次想害他（父亲瞽叟和弟弟象合谋想烧死舜、让舜挖井时活埋舜）。
- **方法论的使用**: 舜没有"念怨"——他没有记住这些伤害，没有在后来进行报复，反而继续孝顺父亲、友爱弟弟。当尧问谁可以继承天下时，舜没有说"我比弟弟强"，而是让弟弟继续得到任用。
- **结论**: "忘怨"不是软弱，而是"不让怨恨成为自己的枷锁"。
- **结果**: 舜成为中华文化的道德楷模，成为"二十四孝"之首，继承了尧的帝位。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. **长期积累的怨恨需要释放**：用户对某人的怨恨已经累积很久，每一次想起来都觉得痛苦，想知道"我该如何处理这份怨恨"。
2. **在关系中评估"谁付出更多"**：用户觉得自己付出很多但对方不领情，想知道"我的付出是否变成了筹码"。
3. **人际冲突后的修复**：用户和某人发生了冲突，事后想修复关系但不知道该怎么处理恩怨。
4. **想放下但放不下**：用户知道应该"算了"，但心里过不去，想找一个框架来真正"放下"。
5. **评估自己的"记账方式"是否健康**：用户想知道自己的人际恩怨观是否正常，是否过度记仇或过度记恩。

### 语言信号 (用户的话里出现这些就应激活)

- "我对他这么好他却这样对我"
- "为什么我总是记得别人对我的伤害"
- "我放不下"
- "他欠我的"
- "我不想原谅他"
- "我帮他这么多，他却..."
- "每次想到那件事我就难受"
- "我该记仇还是算了"

### 与相邻 skill 的区分

- 与 `f08（推己及人对治换位框架）` 的区别：f08 是"通过换位推演来理解对方，从而化解对他的苛责"；f16 是"通过不对称记账来让自己内心更轻松"。f08是理解，f16是放下。
- 与 `f12（一念转魔成佛框架）` 的区别：f12 是"在情绪即将失控的瞬间如何转念"；f16 是"在事后如何处理累积的恩怨"。f12是危机处理，f16是长期心理调适。

## E — 可执行步骤 (Execution)

当 skill 被激活后， agent应按以下步骤执行:

1. **确认当前的"怨恨"或"不甘"**
   - 完成标准: 明确说出"你对谁有怨？或谁欠了你什么？"并量化这个"债"的严重程度（1-10）。
   - 判停条件: 若用户只是泛泛的"我对很多人都有怨"，则要求其具体化到"一个人、一件事"。

2. **执行"不对称记账"检验**
   - 完成标准: 问用户四个问题：①"我对他的付出，我还在念吗？"（如果念，说明你的"功"没有忘）；②"我伤害过他的地方，我记住了吗？"（如果不记，说明你的"过"没有念）；③"他帮过我的地方，我记住了吗？"（如果不记，说明你的"恩"没有记）；④"他伤害过我的地方，我还在记吗？"（如果还在记，说明你的"怨"没有忘）。
   - 判停条件: 若用户发现自己"记功忘过、记怨忘恩"（完全相反），则提示"这正是你痛苦的根源——你在用错误的记账方式让自己成为受害者"。

3. **制定"记账调整"的具体行动**
   - 完成标准: 基于检验结果，制定调整方向：①"从今天起，我对__的付出不再念念不忘"；②"从今天起，我对__的恩惠开始记录"；③"从今天起，我对__的怨恨开始放下"（可以选择"完全放下"或"设定一个放下的时间表"）。
   - 判停条件: 若用户坚持"我不想原谅他"，则尊重其选择，但提示"你不原谅他，受伤的是你自己"。

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **对方的行为是严重的持续性伤害**：如虐待、持续的情感操控，此时"忘怨"可能变成"继续容忍伤害"的理由。需要先保护自己，再谈记账。
- **用户还没有准备好面对自己的"记账问题"**：如果用户只是"想抱怨"而不是"想改变"，过早引入此框架可能让用户感到被评判。
- **涉及法律问题的伤害**：如果伤害涉及法律（欺诈、暴力等），此时"忘怨"不能代替法律手段。需要先处理法律问题，再谈心理调适。

### 作者在书中警告的失败模式

- **以短攻短以顽济顽（ce15）**：当众揭人短处、以愤怒厌恶对待别人的愚顽——这正是"记怨"的极端表现。"忘怨"的对立面不是"原谅"，而是"继续用怨恨来定义你和对方的关系"。
- **伪善诈善之患（ce16）**：伪装善良的伪君子——如果你的"忘怨"是表演性的（表面原谅，内心记仇），这不是真正的"忘怨"，而是更深的自我欺骗。

### 容易混淆的邻近方法论

- **"忘怨" vs "原谅"**：忘怨是"让怨恨不再占据我的内心"；原谅是"承认对方的行为并接受它"。忘怨比原谅更容易做到——你不需要原谅对方，只需要"不再让这件事消耗你"。
- **"记恩" vs "欠债"**：记恩是"我感谢你，我记住你的好"；欠债是"你帮了我，你将来要还"。前者是感恩，后者是筹码。f16要求的是"记恩感恩"不是"记恩欠债"。
