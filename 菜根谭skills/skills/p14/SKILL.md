# 谨言慎行原则

## R — 原文 (Reading)

> "十语九中，未必称奇，一语不中，则愆尤骈集；十谋九成，未必归功，一谋不成，则訾议丛兴。君子所以宁默毋躁，宁拙无巧。"
>
> — 洪应明, 《菜根谭·谨言慎行 君子之道》

---

## I — 方法论骨架 (Interpretation)

1. **说话和谋事的回报是不对称的。** 说对十句没人记得，说错一句怨尤集身；谋对九次没人称赞，谋错一次众议蜂起。
2. **在这个不对称下，最优策略是沉默和拙朴。** 不是说不能说话，而是不要为了表现而多言；不是说不能谋事，而是不要为了邀功而多谋。
3. **"宁默毋躁"——宁可沉默，不要急躁。** 急躁说话容易出错，沉默至少不会说错。
4. **"宁拙无巧"——宁可笨拙，不要耍巧。** 笨拙做事至少扎实，耍巧容易露馅。
5. **核心：减少"被评价"的次数，就是减少"被负面评价"的概率。** 不说话、不多谋，就不会有失败可被评价。

---

## A1 — 书中的应用 (Past Application)

### 案例: 洪应明论"十语九中一语不中则愆尤骈集"

- **问题**: 为什么做了很多、说了很多，却换来负面评价？
- **方法论的使用**: 洪应明指出这个不对称性——正面贡献不被记得，负面失误被放大
- **结论**: 君子选择"宁默毋躁，宁拙无巧"——通过减少失败概率来保护声誉
- **结果**: 少说少错，多说多错；少谋少败，多谋多败

---

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. **说完话后担心是不是说多了/说错了** — "我是不是不该说这个？"、"我这样说会不会让人误会？"
2. **准备发言/演讲，担心说不好时** — "我不知道该说什么"、"万一我说错了怎么办"
3. **做决策前犹豫要不要再想想** — "我是不是太冲动了？"、"我是不是考虑得太少了？"
4. **看到别人说错话/做错决定，觉得自己也要小心时** — "我以后也要谨慎点"、"言多必失"

### 语言信号 (用户的话里出现这些就应激活)

- "我是不是说多了？"
- "我会不会说错话？"
- "我是不是太冲动了？"
- "我还没想好该怎么说"
- "言多必失啊"
- "我说了会不会有问题？"

### 与相邻 skill 的区分

- 与 `p13中和为福偏激为灾` 的区别: p13是整体性格（不要太急躁），p14是具体行为（说话和谋事要谨慎）；p13是战略，p14是战术
- 与 `p20放下屠刀` 的区别: p20是情绪控制（怒火时猛然转念），p14是日常行为（说话前先想想）；p20是应急反应，p14是长期习惯

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **识别用户当前是要"说"还是要"做"** — 说话前还是说话后？谋事前还是谋事后？
   - 完成标准: 用户明确自己的情境

2. **说话前：用"十语九中"测试——这句话说出来，十个人里有几个会记住？有一个说错的后果是什么？**
   - 完成标准: 用户权衡说话的风险和收益

3. **说话后/做错后：用"宁默毋躁"测试——现在补救会越描越黑，还是沉默更好？**
   - 完成标准: 用户决定是补救还是沉默

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **需要清晰表达观点的场景**：如重要谈判、汇报、领导力展现，此时过度沉默可能被认为没有立场
- **创意发散阶段**：需要大量试错和讨论，不能因为"怕说错"而不说
- **关系中的真诚沟通**：有时需要"说错话"才能表达真实感受

### 作者在书中警告的失败模式

- 谨言慎行不及（来自反例ce07）：说话不假思索脱口而出，计谋过度患得患失，总想表现自己的聪明
- 机制：话说多错多，十句对九句无人称赞，一句不中则怨尤集身

### 作者的盲点 / 时代局限

- 现代职场强调"存在感"和"声音"，过度沉默可能影响职业发展
- 某些组织文化鼓励"大胆发言"，此时"宁默毋躁"可能被视为消极

### 容易混淆的邻近方法论

- **中和为福(p13)**：p13更强调性格层面的平衡，p14更强调说话和谋事的行为层面
- **心地光明(p03)**：p03是"心地要光明"，p14是"说话要谨慎"——p03要求透明，p14要求收敛
