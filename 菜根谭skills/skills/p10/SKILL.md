# 无过是功无怨是德原则

## R — 原文 (Reading)

> "处世不必邀功，无过便是功；与人不求感德，无怨便是德。"
>
> — 洪应明, 《菜根谭·无过是功 无怨是德》

---

## I — 方法论骨架 (Interpretation)

1. **功不是"做出来了"，而是"没做错"。** 洪应明认为，不邀功反而有功——因为不出错就是最大的功。
2. **德不是"施恩了"，而是"没招怨"。** 不求回报的施恩，如果最后没让人怨恨，就是最大的德。
3. **邀功的人实际上在做什么？** 在把自己的价值绑定在别人的认可上——别人不认，就觉得自己没价值。
4. **求感德的人实际上在做什么？** 在做一个隐形的交易——我给你恩，你欠我情；别人不感恩，就觉得被亏欠了。
5. **核心：减少对外部评价的依赖。** 无过=不邀功，无怨=不求德——两者都是在说：你的价值不需要别人来定义。

---

## A1 — 书中的应用 (Past Application)

### 案例: 洪应明论"不邀功"的悖论

- **问题**: 为什么做了很多事却得不到认可？要不要去"争取功劳"？
- **方法论的使用**: 洪应明提出一个反直觉的观点：不要去邀功，因为"无过便是功"——不出错才是最大的功劳
- **结论**: 邀功者往往因为"有所求"而显得动机不纯，反而得不到真心认可；不邀功者因为无所求而显得纯粹，反而赢得尊重
- **结果**: 减少对外部认可的依赖，反而能更坦然地面对评价

---

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. **做了大量工作但得不到认可，纠结要不要"让领导知道"时** — "我做了这么多，为什么没人看到？"、"我要不要去争取一下功劳？"
2. **付出后没有回报，感到心理不平衡时** — "我对他这么好，他怎么这样对我？"、"我帮了这么多，为什么没人感激？"
3. **在"默默做事"和"适度表现"之间纠结时** — "我该不该让自己的付出被看到？"、"太老实是不是会吃亏？"

### 语言信号 (用户的话里出现这些就应激活)

- "我做了这么多没人看到"
- "凭什么功劳是别人的？"
- "我这么辛苦为什么没人心疼"
- "我帮了他，他怎么这样对我？"
- "我要不要让他们知道我做了什么？"

### 与相邻 skill 的区分

- 与 `p03心地光明才华韫藏` 的区别: p03是"才华要藏"，p10是"功劳也要藏"；p03针对能力，p10针对功劳
- 与 `p07忍让为高利人利己` 的区别: p07是主动让步，p10是"不邀功不求德"——p07是行动策略，p10是心态策略

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **识别用户是否在"邀功"或"求德"——内心有个声音在说"我做了这么多，应该被认可"**
   - 完成标准: 用户意识到自己在等待外部认可

2. **用"无过便是功"测试：如果你做的事不出错，不需要别人认可，它的价值是什么？**
   - 完成标准: 用户找到不依赖外部评价的自我价值感

3. **用"无怨便是德"测试：如果你帮助了别人，但你不期待回报，你是在做真正的善事吗？**
   - 完成标准: 用户区分"真正的善意"和"隐形的交易"

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **需要明确绩效评估的组织**：过度"无过便是功"会让有能力的人变得保守不前
- **职场晋升关键节点**：此时需要适度展示成果，过度低调可能错过机会
- **对方故意贬低/打压你时**：此时需要适当地维护自己的价值，不是完全无过便是功

### 作者在书中警告的失败模式

- 器小禄薄（来自反例ce14）：处处与人计较，只能占便宜不能吃亏，做决定只看短期
- 机制：计较功劳的人，禄薄泽短，因为没有人愿意与只计较小利的人合作

### 作者的盲点 / 时代局限

- 现代职场有完整的绩效评估体系，"无过便是功"可能导致被动不前
- 某些"邀功"是必要的职业保护，不让领导知道工作成果可能被视为没有贡献

### 容易混淆的邻近方法论

- **谦逊低调(p27)**：p27更强调成功后要收敛；p10是日常心态——无论成功失败都不邀功
- **淡泊明志(p05)**：p05是"志"要淡，p10是"功"要淡；p05更根本
