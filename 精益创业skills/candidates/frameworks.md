- id: f01
  title: 开发—测量—认知循环 (Build-Measure-Learn)
  type: framework
  source_chapter: 引言+第五章
  source_quote: |
    "新创企业的基本活动是把点子转化为产品，衡量顾客的反馈，然后认识到是应该改弦更张还是坚守不移。所有成功的新创企业的流程步骤都应该以加速这个反馈循环为宗旨。"
  summary: |
    将点子转化为产品(开发) → 衡量顾客反馈(测量) → 认知应该坚持还是转型 → 再循环。
    核心不是速度，而是"加速反馈循环"。每个活动都要问：它是否加速了这个循环？
  tags: [core-loop, feedback, iteration]

- id: f02
  title: 信念飞跃假设检验
  type: framework
  source_chapter: 第五章
  source_quote: |
    "每个商业计划、商业模式、愿景，最终都要被拆解成一系列可以被快速检验的假设。"
  summary: |
    把宏大的战略/愿景分解为具体的、可被验证的假设（信念飞跃）。每个假设都是一个实验。
    例如：顾客会对X感兴趣、顾客会愿意付费Y元、顾客会完成Z行为。
  tags: [hypothesis, strategy, validation]

  type: framework

- id: f04

  type: framework
  source_chapter: 第八章
  source_quote: |
    "转型是一种有条理的方向性改变，用来测试新的产品、战略和增长引擎的基础假设。"

    当创新核算显示进展停滞时，系统性地评估10种转型类型：
    放大/缩小转型、客户细分/需求转型、平台/商业架构转型、价值获取/增长引擎/渠道/技术转型。
    每种转型都是对基本假设的重新检验。
  tags: [pivot, decision, strategy]

- id: f05
  title: 同期群分析
  type: framework
  source_chapter: 第七章
  source_quote: |
    "同期群分析把复杂的顾客行为转化为以人为本的报告。每个同期群分析都会说：在这段时间内使用了我们产品的顾客中，有多少人表现出了我们期望的各种行为。"
  summary: |
    按顾客获得时间分组追踪，而非看总用户数。关注每个同期群的行为：激活率、保留率、付费率。
    避免被"总注册量"等虚荣指标误导，看清真实的顾客行为趋势。
  tags: [cohort-analysis, metrics, customer-behavior]

- id: f06
  title: 小批量原则
  type: framework
  source_chapter: 第九章
  source_quote: |
    "小批量生产的方法比较出色...用大批量的话，我们一直要到接近流程终点才会发现问题。而用小批量的话，我们几乎能马上发现问题。"
  summary: |
    把大批量的工作拆成小批次，每次只处理一件。好处：
    1）快速发现错误（不用等到最后）2）减少返工成本 3）加速反馈循环。
    在产品开发中对应"最小化可行产品"而非完美产品。
  tags: [small-batch, lean, speed]

- id: f07
  title: 增长引擎驱动
  type: framework
  source_chapter: 第十章
  source_quote: |
    "可持续的增长来自三种增长引擎中的一种：黏着式增长、病毒式增长和付费式增长。"
  summary: |
    三种引擎：黏着式（顾客主动回来）、病毒式（顾客主动传播）、付费式（广告/销售驱动）。
    每种引擎有不同关键指标。新创企业要搞清楚自己的引擎是哪一种，集中资源优化它。
  tags: [growth-engine, metrics, strategy]

- id: f08
  title: 五个为什么分析
  type: framework
  source_chapter: 第十一章
  source_quote: |
    "五个为什么是一种根源分析方法，通过不断追问'为什么'来找到问题的根本原因，而非仅仅解决表面症状。"
  summary: |
    当问题出现时，连续问5个"为什么"追溯根本原因。按比例投入资源（大问题大投入，小问题小投入）。
    防止组织用短期补丁掩盖深层问题，建立真正的学习型组织。
  tags: [root-cause, problem-solving, continuous-improvement]

- id: f10
  title: 对比测试（A/B测试）
  type: framework
  source_chapter: 第七章
  source_quote: |
    "对比测试就是在同一时间向顾客提供不同版本的产品。通过观察两组人的行为变化，对不同版本的影响力得出推断。"
  summary: |
    同期进行两个版本的实验，控制其他变量，看哪个版本带来期望的行为变化。
    让数据说话而非直觉或职位高低决定产品方向。建立"证据驱动"的决策文化。
  tags: [ab-test, experiment, data-driven]

- id: f11
  title: 三个"可"衡量指标
  type: framework
  source_chapter: 第七章
  source_quote: |
    "三个'可'的衡量指标，即：可执行、可使用和可审查指标。"
  summary: |
    好指标必须满足：可执行（清楚显示因果关系）、可使用（团队能理解并用做决策依据）、
    可审查（任何员工都能验证数据真实性）。不符合这三点的都是虚荣指标。
  tags: [metrics, measurement, accountability]

- id: f12
  title: 创新沙盒
  type: framework
  source_chapter: 第十二章
  source_quote: |
    "创新沙盒是指在大组织内部划定的受保护实验空间，让内部创业团队可以在不受正常流程约束的情况下快速实验。"
  summary: |
    大企业无法像新创企业一样敏捷，但可以通过"沙盒"划定保护区域：
    给创新团队独立资源、独立权限，用不同的成功标准衡量，保护颠覆式创新不被主流流程扼杀。
  tags: [innovation, corporate, sandbox]
