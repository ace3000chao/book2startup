# Skill: 知彼知己框架

## R — 原文引用
> "故曰：知彼知己，百战不殆。不知彼而知己，一胜一负。不知彼不知己，每战必败。"
> ——孙子·谋攻第三

## I — 方法论骨架

**"知彼知己"三层次决策模型：**

| 认知层次 | 胜率 | 行动策略 |
|---------|------|---------|
| 知彼又知己 | 必胜（不殆） | 主动出击，精准决策 |
| 不知彼但知己 | 胜负各半（50%） | 谨慎试探，先守后攻 |
| 不知彼又不知己 | 每战必败 | 禁止行动，先调研 |

**核心洞察：**
- "彼"（对手/外部环境）的信息比"己"更难获取
- 不知彼的风险 > 不知己的风险
- "知己"是底线，"知彼"是加分项

## A1 — 书中案例
孙武在用间第十三中详述"先知"的重要性：
- 伊挚（伊尹）曾卧底夏朝，了解夏桀暴政，为商汤伐夏提供情报
- 吕牙（姜子牙）在殷商为官，熟知商纣内部情况，助周武王灭商
- 结论："以上智为间者，必成大功"

## A2 — 未来触发情境
- 🎯 创业者在推出新产品前，不确定竞品动向
- 🎯 商业谈判前，不了解对方底线和真实诉求
- 🎯 职业转型时，不清楚目标行业/岗位的真实需求
- 🎯 投资决策前，不了解标的公司实际经营状况
- 🎯 团队扩张前，不了解候选人的真实能力和动机

## E — 执行步骤

### 步骤1：画"彼己矩阵"
列出决策涉及的"彼"（外部变量）和"己"（内部变量）

```
        已 知    未 知
彼     ___________
      |           |
      |           |
己     ___________
      |           |
      |           |
```

### 步骤2：识别"不知彼"区域
针对"彼"中的未知项，制定情报收集计划：
- 直接调研：访谈、问卷、实地考察
- 间接调研：公开数据、行业报告、竞品分析
- 反推验证：通过对方行为倒推其真实意图

### 步骤3：决策树映射
根据彼己矩阵，选择行动策略：
| 区域 | 策略 |
|-----|------|
| 知己+知彼 | 全力出击 |
| 知己+不知彼 | 谨慎试探，设置止损 |
| 不知彼+不知己 | 禁止行动，先建情报网 |

## B — 边界（什么时候不适用）

❌ **边界1：** 高度不确定性的探索期
- 当你处于"0到1"阶段，对手和市场都不存在时
- 此时"知彼"的对象尚未形成，过度分析反而贻误战机

❌ **边界2：** 速度优先的闪电战
- 当行业窗口期极短（如热点营销、爆款时机）
- 此时信息不完整也必须行动，"先开枪再瞄准"

❌ **边界3：** 创业新人（经验不足）
- 如果"知己"能力都不具备（自我认知不清）
- 先建立自我认知体系，再谈知彼
