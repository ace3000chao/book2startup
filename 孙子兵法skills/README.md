## ✨ 效果示例

### 使用前 vs 使用后

**场景：** 大公司要进入你的市场，你该怎么办？

**使用前（普通AI）：**
> "要保持冷静，分析竞争格局，提升产品竞争力..."

**使用"避实击虚"Skill后：**
> **避实击虚 · 框架分析**
>
> 核心洞察：水之形，避高而趋下；兵之形，避实而击虚。
>
> **三步走策略：**
> 1. **画"虚实地图"** — 对手重兵投入的地方（实）vs 忽视的地方（虚）
> 2. **找"一卒可胜"** — 在"虚"中选择一个足够锋利的切入点
> 3. **建立"敌变我变"机制** — 保持敏捷，不在旧战场恋战
>
> **案例参考：** 拼多多崛起——阿里京东主攻"消费升级"（实），拼多多选择"下沉市场"（虚）

### 实际对话示例

```
你：你是一个创业CEO，竞品刚融了5000万要碾压你，用孙子兵法的智慧给我建议。

AI（调用"致人而不致于人"Skill）：

## 🎯 核心问题：你在被动应对

孙子曰："善战者，致人而不致于人"

你现在被对手调动——他们在牵着你的鼻子走。

## 🔄 夺回主动权三步

**步骤1：诊断"致于人"现状**
- 你现在被谁调动？_____
- 你现在"不得不"做什么？_____
- 如果继续这样，最坏结果是什么？_____

**步骤2：找"致人"的杠杆点**
- 你有什么对手想要但稀缺的？
- 你能攻击对手什么不可承受之害？
- 你能创造什么"不来会后悔"的利益场景？

**步骤3：设计"利之"行动**
```
对手观察到的信号 → 对手判断"有利" → 对手主动行动 → 我方以逸待劳
```

## ⚠️ 边界提醒
- 如果实力悬殊，"利之"手段可能被识破
- 如果需要快速建立信任，过度操控会损害长期关系
```

## ✨ 应用案例

问：我的学生正在创业“动听喵AI社交互动KTV”项目（轻量化自助连锁KTV），请你使用孙子兵法skills给一些具体指导。

答：（我的openclaw的创业导师Agent回答）

### 《孙子兵法》视角下的动听喵连锁扩张战略指导

> 适用场景：从 2 家店扩张到 30-50 家连锁的品牌增长战役
> 指导对象：动听喵 AI 社交互动 KTV 创业团队

---

#### 战略总纲

**"先为不可胜，以待敌之可胜"** —— 扩张之前，先把"不可胜"的能力建好。

---

#### ① 扩张之前，先问自己"不可胜"在哪里

《孙子兵法》说"先为不可胜，以待敌之可胜"——

你现在两家店，生意不错。但"不可胜"是你的护城河，不是你的流水账。

❓ **你的"不可胜"是什么？**

| 壁垒维度 | 具体内容 |
|---------|---------|
| 技术壁垒 | 那套 100% 全自研的 AI 系统？（别人复制不了） |
| 模式壁垒 | "自助+社交"的差异化模式？（年轻人就是认这个） |
| 成本壁垒 | 比传统 KTV 低 75% 的成本结构？（这就是你的杀手锏） |

👉 **扩张之前，先把这两家店的"核心壁垒"写成文档、形成标准、让每个店长都能复制**

---

#### ② 开第三家店，别只算财务账

《孙子兵法》说"杂于利害"——

**利的方面：** 账算得过来，60-100 万，8-16 个月回本

**害的方面：**

- 你的人才梯队跟得上吗？（店长够不够用？）
- 系统能支撑 30 家吗？（现在就要考虑技术架构的扩展性）
- 管理半径超过 5 家之后，你有什么机制？

👉 **开第三家之前，先画一张"能力短板图"——告诉团队：哪里还不能复制，哪里必须先补上**

---

#### ③ 开店要"势险节短"，不要慢慢悠悠

《孙子兵法》说"其势险，其节短"——

- **势险** = 开店之前，把势能蓄足（位置选好、流量谈好、团队培训好）
- **节短** = 开业之后，快速打出名声（第一周就要有动静，不能静悄悄开业静悄悄死）

👉 **第三家店开业策略：前 30 天集中火力做一件事——让周边 5 公里内的人都知道"动听喵是什么"**

---

#### ④ 团队士气是"扩张期"最大的敌人

《孙子兵法》说"三军可夺气，将军可夺心"——

开了两家店，团队开始有"小富即安"的心态：

- 老板觉得够了，不想那么累了
- 店长觉得管两家店已经很忙了，不想再增加

**这是最危险的时候。**

👉 **创始人自己要保持"锐气"——你的状态就是整个团队的天花板。如果你松了，整个组织就松了**

---

#### ⑤ 珠三角 30-50 家的目标，怎么拆解？

《孙子兵法》"致人而不致于人"——

→ 让加盟商/合伙人觉得"这个机会不抓住会后悔"

→ 你的品牌势能在上升期，此时扩张最省力

→ 等别家先做起来，你就被动了

👉 **2026 年的核心动作：把"2 家→30 家"的第一步"样板验证"做好**

- 数据说话
- 投资回报率说话
- 老店运营指标说话

**有了样板，后面的加盟商自己会来。**

---

#### ⑥ "五事七计"自检清单

开下一家店之前，用这七个问题自检：

| 七计问题 | 自检要点 |
|---------|---------|
| **主孰有道** | 你的愿景和使命，团队认同吗？加盟商认同吗？ |
| **将孰有能** | 你的店长储备够不够？谁可以独当一面？ |
| **天地孰得** | 选址数据模型跑通了吗？珠三角哪几个城市先布点？ |
| **法令孰行** | SOP 标准操作手册每家店都一样吗？ |
| **兵众孰强** | 团队整体战斗力排在行业什么位置？ |
| **士卒孰练** | 新店长培训周期是多长？能否快速复制？ |
| **赏罚孰明** | 激励机制清晰吗？店长和总部的利益绑定机制是什么？ |

---

#### 现在最该做的 3 件事（按优先级）

| 优先级 | 行动 | 时间节点 |
|-------|------|---------|
| 1️⃣ | 把两家店的运营数据做成一份"样板手册"（选址模型、定价数据、客流规律、AI 系统效率） | **本周** |
| 2️⃣ | 物色第三家店店长，开始带教（人比店更重要） | **本月** |
| 3️⃣ | 测试"加盟合伙人"模式的第一版合同（先把机制跑通） | **本季** |

---

#### 最后一句话

> 开两家店是"生意"，开 30 家是"企业"。
>
> 你现在最需要的不是更多店，而是**一套能让 30 家店都用同一套逻辑运转的系统**。
> 系统建好了，扩张是自然的结果。

### Skill文件说明

每个Skill包含两部分：

| 文件 | 说明 | 示例 |
|------|------|------|
| `SKILL.md` | Skill定义 | 包含R/I/A1/A2/E/B六段 |
| `test-prompts.json` | 测试用例 | 验证Skill是否正确调用 |

#### SKILL.md 结构解读

每个SKILL.md包含：

- **description** — 触发描述（什么情况下调用）
- **R（原文引用）** — 孙子原话
- **I（方法论骨架）** — 核心逻辑和框架
- **A1（书中案例）** — 原文中的应用
- **A2（触发情境）** — 你在什么场景下会用到
- **E（执行步骤）** — 1-2-3可执行步骤
- **B（边界）** — 什么时候不适用

## 🧠 这些Skill是怎么生成出来的

本仓库使用 **book2skill** 方法论，从《孙子兵法》原文中系统化蒸馏Skills。

### 流水线概览

```
阶段0: 整书理解 → BOOK_OVERVIEW.md
阶段1: 5个提取器并行提取 → 候选池（39框架+67原则）
阶段1.5: 三重验证筛选 → 通过的候选
阶段2: RIA++构造Skill → 18个SKILL.md
阶段3: Zettelkasten链接 → INDEX.md
阶段4: 压力测试 → test-prompts.json
```

### 三重验证标准

每个候选必须通过：

| 验证 | 问题 | 通过条件 |
|------|------|---------|
| **V1 跨域** | 书中至少有2个独立段落有佐证？ | ✓ |
| **V2 预测力** | 能用它回答书里没明说的新问题？ | ✓ |
| **V3 独特性** | 不是任何聪明人都会说的常识？ | ✓ |

### RIA++结构

每个Skill都包含六段：

| 段落 | 含义 | 作用 |
|------|------|------|
| **R** | Reading（原文引用） | 权威依据 |
| **I** | Interpretation（方法论骨架） | 核心逻辑 |
| **A1** | 书中案例 | 原文应用 |
| **A2** | 未来触发情境 | 何时调用 |
| **E** | Execution（执行步骤） | 如何使用 |
| **B** | Boundary（边界） | 何时不用 |
