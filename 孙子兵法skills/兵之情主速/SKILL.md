# Skill: 兵之情主速框架

## R — 原文引用
> "兵之情主速，乘人之不及，由不虞之道，攻其所不戒也。"
> ——孙子·军争第七 / 九地第十一

## I — 方法论骨架

**速的三层含义：**

| 层次 | 含义 | 商业对应 |
|------|------|---------|
| **速决策** | 快速判断，不犹豫 | 决策速度 |
| **速行动** | 快速执行，抢占先机 | 执行速度 |
| **速变化** | 快速迭代，因敌调整 | 响应速度 |

**速的三大优势：**
1. **敌人来不及准备** — 在对手反应过来之前完成布局
2. **敌人来不及调整** — 趁对手改变策略之前扩大战果
3. **敌人心理崩溃** — 快速连续的打击让对手"来不及思考"

**核心公式：**
```
速度 = 先发优势 + 信息优势 + 心理优势
```

## A1 — 书中案例

孙武举例：
- "后人发，先人至" — 比敌人晚出发，却比敌人先到达
- 关键在于"由不虞之道" — 走敌人意想不到的路

**商业案例：** 字节跳动的"大力出奇迹"
- 快速上线、快速迭代、快速试错
- 抖音vs微视：抖音先发三个月，靠速度建立用户习惯壁垒

## A2 — 未来触发情境
- 🎯 行业出现新风口，需要快速入局时
- 🎯 竞品刚发布新产品，需要快速回应时
- 🎯 融资窗口期短暂，需要快速决策时
- 🎯 市场瞬息万变，需要快速调整策略时
- 🎯 团队效率低下，需要提速时

## E — 执行步骤

### 步骤1：识别"时间窗口"
```
窗口长度 = 竞争对手反应过来需要的时间
窗口价值 = 窗口期能建立的壁垒价值
```

### 步骤2：设计"加速"方案
- 减少审批流程
- 并行处理多个任务
- 授权一线快速决策
- 资源优先集中

### 步骤3：找"不虞之道"
- 竞争对手没想到的路径
- 竞争对手不屑于做的细分市场
- 竞争对手复制不了的能力

### 步骤4：建立"持续速度"机制
- 速战只是一时
- 需要建立"快"的组织能力
- 文化和流程都要支持"快"

## B — 边界（什么时候不适用）

❌ **边界1：** 需要深度思考的决策
- 重大战略、技术架构需要充分论证
- "快决策"可能带来"错决策"

❌ **边界2：** 高风险行动
- 速度越快，错误成本越高
- 涉及大量资源时，谨慎优于速度

❌ **边界3：** 客户关系维护
- 快速响应≠过度打扰
- 过度追求速度会牺牲质量

❌ **边界4：** 差异化构建期
- 有时候"慢"反而是壁垒
- 打磨产品比快速推出更重要
